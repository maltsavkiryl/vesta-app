# Designing Native Mobile Apps

Native mobile product-design guidance for teams building React Native apps that still need strong iOS- and Android-specific UX judgment.

## Example commands

- Review this screen for iOS-native interaction and layout issues.
- Audit this Android flow against Material 3 and platform expectations.
- Suggest React Native UI changes that make this feature feel more native on both platforms.

## Skills included

- [iOS Design Guidelines](https://skills.sh/ehmo/platform-design-skills/ios-design-guidelines)
  Covers Apple Human Interface Guidelines for iPhone surfaces, including safe areas, touch targets, Dynamic Type, Dark Mode, native navigation, and common anti-patterns.
- [Android Design Guidelines](https://skills.sh/ehmo/platform-design-skills/android-design-guidelines)
  Covers Material Design 3 and Android platform rules, including dynamic color, adaptive layouts, navigation patterns, accessibility, and edge-to-edge behavior.
- [React Native Design](https://skills.sh/wshobson/agents/react-native-design)
  Covers React Native styling, navigation, gestures, and Reanimated-based interaction patterns for cross-platform product UI work.

## Notes

This plugin complements the existing `building-react-native-apps` plugin in this repository.

- Use `building-react-native-apps` for React Native implementation, performance, and upgrade work.
- Use this plugin when the main question is native product design quality, platform conventions, or UI/UX fit on iOS and Android.
